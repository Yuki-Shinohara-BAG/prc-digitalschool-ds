# 研修FAQ：PPH売上向上分析（データ分析開始前用）

講師の手元に置いて、聞かれた時にサッと答えられるように。

## このFAQの前提

- 参加者が見られるデータは `データ仕様書.xlsx` と CSV データのみ
- `machine_*.csv` は配布しない方針（仕様書 シリアルNo命名規則 シートで代替）
- **本FAQはデータを実際に触る前** の質問集。分析結果や手法に関する質問は別資料（研修Notebook・講師ガイド）参照
- 公式情報の補完リンク：
  - 製品ラインナップ：<https://www.konicaminolta.jp/business/products/graphic/ondemand_print/index.html>
  - オプション詳細（C14010 シリーズ ユーザーズガイド）：<https://www.btapac.konicaminolta.com/users_guide/accuriopress_C14010_series/1_1_0/jp/index.html>
- 答えに迷ったら「データ仕様書を見てみよう」「公式ページで確認してみよう」「社内の○○に確認します」のいずれかで返してOK

## A. データ全体に関する質問

### Q1. データは全部で何ファイル？

A. **CSVが3つ + Excel仕様書1つ**

| ファイル | 内容 |
| --- | --- |
| `metadata_*_202401.csv` | 2024年1月の累積カウンタ・走行距離 |
| `metadata_*_202501.csv` | 2025年1月の同上 |
| `option_*_202401.csv` | 各機体のオプション搭載状況 |
| `データ仕様書.xlsx` | 列定義／オプション意味／センターID／シリアルNo命名規則 |

### Q2. 機種コード（model_cd）はどこから取得？

A. **メタデータCSVに `MODEL_CD` 列があります**。データ仕様書の「メタデータ」シート参照。

### Q3. 開発名（Cypress等）はどこ？

A. **データ仕様書の「シリアルNo命名規則」シート**を参照。mach_idの先頭4文字 → 開発名 → model_cd の対応表があります。

### Q4. 機体（mach_id）の単位で考えればOK？

A. その通り。**1台の機械 = 1つの mach_id**。全分析は mach_id を主キーに進めます。

## B. 列・データの意味

### Q5. `pmtc_counter` は何？単位は？

A. **累積メンテナンスカウンター（印刷枚数）**、単位は **枚**。
データ仕様書「メタデータ」シートに「累積メンテナンスカウンター(枚数)」と明記。
2024年1月時点と2025年1月時点の **差分** が「1年間の印刷枚数」になります。

### Q6. `pmtc_distance` は何？単位は？

A. **累積メンテナンスカウンター（距離）**。
推定単位は ** m**。「機械が物理的にどれだけ動いたか」の指標として使えます。

### Q7. `create_date` は何の日付？設置日？

A. データ仕様書では「作成日」とだけ記載。**レコードが作成された日付**（多分カウンタ取得日）。
**機械の設置日とは別** 。

### Q8. `csrc_cntr_id` は何？

A. **CSRCセンターID**。販売・サービス拠点の識別子。
データ仕様書「CSRC センターID_一覧」シートで国・地域に紐づけ可能。

## C. modelに関する質問 ⭐よく出る

### Q9. modelごとに何が違うの？

A. 主に以下が違います：

| 観点 | 違い |
| --- | --- |
| **印刷スピード** | プロダクション機（PRESS系）は速い、エントリー機（C2060等）はゆっくり |
| **対応用紙サイズ・坪量** | 上位機ほど厚紙・特殊紙対応 |
| **カラー/モノクロ** | C系はカラー、PRO系もカラー、HC系は高画質モデル |
| **想定用途** | プロダクション印刷、オフィス印刷、ラベル印刷など |
| **オプション拡張性** | 大型機ほど多種オプション接続可能 |

詳細は公式の製品ラインナップで確認：<https://www.konicaminolta.jp/business/products/graphic/ondemand_print/index.html>

### Q10. ハイエンド機・ミドル・エントリーの区分は？

A. **製品スペック（特に1分当たりの印刷枚数 = PPM）** で区分されます：

| 区分 | 例 | 特徴 |
| --- | --- | --- |
| プロダクション/ハイエンド | C14000, C12000, PRESS6136, PRESS6120 | PPMが高い、対応用紙・坪量の幅が広い |
| ミドル | C6085, C6100, C7090, PRO1100 | 中速、汎用業務向け |
| エントリー | C2060, C3070, C4065 | PPMが低い、オフィス用途中心 |
| ラベル機（特殊） | LABEL230 | ラベル印刷専用 |

詳細スペックは公式ラインナップで確認：<https://www.konicaminolta.jp/business/products/graphic/ondemand_print/index.html>

### Q11. 「PRESS系」と「C系」の違いは？

A.
- **PRESS系（PRESS6120/6136/6272P等）**: **モノクロ機**
- **C系（C3070, C4080等）**: **カラー機**
- **PRO系（PRO1100）**: 旧世代プロダクション機
- **HC系（C73HC, C83HC等）**: 高画質カラーモデル（High Color）

### Q12. 「Cypress」「Rosewood」って何？

A. **開発名（コードネーム）**。商品名（C14000等）とは別の社内呼称。データ仕様書「シリアルNo命名規則」シート参照。

| 開発名 | 主な機種 |
| --- | --- |
| Cypress | C14000, C12000 |
| Rosewood | C4065, C4070, C4080 |
| Maple | C7100, C7090 |
| Chagall-MLK | C6085, C6100 |
| Degas-eMLK | C3070, C3080 |
| Degas-e | C2060, C2070 |
| Degas | C1060, C1070 |
| Chagall | C1085, C1100 |
| SymphonyM1 | PRESS6120, PRESS6136 |
| SymphonyMe | PRESS7120, PRESS7136 |
| Symphony3M | PRO1100 |
| Symphony3Me | Print2100 |
| Degas-eMLKPV | C83HC |
| Degas-eMLKR2R | LABEL230 |
| CypressR2R | AL400 |

### Q13. 最新機種はどれ？

A. **公式ラインナップで「NEW」マークが付いているもの**が最新機種です。
確認はこちら：<https://www.konicaminolta.jp/business/products/graphic/ondemand_print/index.html>

### Q14. C3070 と C4080 の違い？

A. **マシンスペック（印刷スピード・対応用紙・オプション拡張性等）** が違います。
- **C4080（Rosewood1）**: 上位機。PPMが高く、対応用紙の幅が広く、オプション拡張性も高い
- **C3070（Degas-eMLK）**: 下位機。中速で汎用業務向け
- 同じ「C」シリーズでも世代・スピード・対応用紙が異なります
- 詳細スペックは公式ラインナップで確認：<https://www.konicaminolta.jp/business/products/graphic/ondemand_print/index.html>

### Q15. (要確認) の機種があるけど何？

A. データ仕様書の「シリアルNo命名規則」シートで開発名が `(要確認)` になっている3機種：
- **C73HC**（260台）
- **C71HC**（232台）
- **C71CF**（38台）

→ 全て HC系（High Color、高画質）または CF系。社内の商品企画に確認お願いします。

## D. オプションに関する質問 ⭐よく出る

### Q16. ADF・PFU・LCT・Bypassって何？

A. データ仕様書「オプションデータテーブル」シートに各列の意味があります。よく出るやつ：

| 略称 | 正式名 | 意味 |
| --- | --- | --- |
| ADF | Auto Document Feeder | 自動原稿送り装置（原稿を連続で読み込む） |
| PFU | Paper Feed Unit | 給紙ユニット（用紙を給紙する装置） |
| LCT | Large Capacity Tray | 大容量給紙トレイ |
| Bypass | — | 手差しトレイ |
| Stacker | — | 排紙積載機 |
| Saddle stitcher | — | 中綴じ製本ユニット |
| Perfect binder | — | 無線綴じ製本ユニット |
| Multi puncher | — | 多孔パンチユニット |
| Transporter | — | 用紙搬送ユニット（機種間を繋ぐ） |
| Finisher | — | 仕上げユニット（ステープル・パンチ等） |

詳細：<https://www.btapac.konicaminolta.com/users_guide/accuriopress_C14010_series/1_1_0/jp/index.html>

### Q17. PFU1 / PFU2 / PFU3 の違いは？

A. **接続位置（給紙段の順序）の違い**：
- **PFU1**: 1段目の追加給紙ユニット
- **PFU2**: 2段目の追加給紙ユニット
- **PFU3**: 3段目の追加給紙ユニット

機種ごとに最大何段まで増設できるかが決まっています。詳細はユーザーズガイド参照。

### Q18. Stacker1 / Stacker2 / Transporter1〜6 の違い？

A. **接続位置の違い**（直後/中間/最後など）。詳細はユーザーズガイド参照。
一般的には：
- 番号小（1）：機体に近い側、よく使われる
- 番号大（4〜6）：複数オプションを連結する際の追加位置、レア

### Q19. 高機能フィニッシャーって具体的に何？

A. データ仕様書から、以下のような **製本・仕上げ系オプション**：
- Saddle stitcher（中綴じ）：SD-506, SD-513
- Perfect binder（無線綴じ）：PB-503
- Multi puncher（多孔パンチ）：GP-501
- All-in-One Finisher：FS-532, FS-541

これらは **プロダクション用途（冊子作り・本作り）** で使われます。

### Q20. オプション同士に依存関係ある？

A. はい。**機種ごとに付けられるオプションが決まっています**。
例えばフィニッシャー系を付ける場合は所定のTransporter（搬送ユニット）が必要、といったルールがあります。詳細はユーザーズガイド参照。

### Q21. Ring Binder が搭載数0なのはなぜ？

A. データ仕様書「オプションデータテーブル」シートを見ると、Ring Binder（OPTION_ITEM19）の値は **「(搭載なし)」** と記載されています。
製品ラインナップとしては存在しますが、本データの機体には搭載例がない、という意味です。

## E. 地域・販社に関する質問

### Q22. CSRCって何？

A. 販社/サービスが確認できるデータサービスの名称。

### Q23. 「地域」「販社」はどこから取れる？

A. **データ仕様書「CSRC センターID_一覧」シート**から `csrc_cntr_id` で結合：
- Country_ISO3166：国名（ISO 3166準拠の英語名）
- 国名_ISO3166：日本語の国名
- 地域：「北アメリカ」「東アジア」など

### Q24. 中国は東アジア？南アジア？

A. **東アジア**。CSRCセンターID一覧で確認できます。
ちなみに地域区分：「東アジア」「北アメリカ」「南アジア」「西ヨーロッパ」「東ヨーロッパ」「東南アジア」「オセアニア」「中東」「ロシア」「南アフリカ」「南アメリカ」「北ヨーロッパ」「中央アフリカ」「中央アジア」「中央アメリカ」

### Q25. 同じ国でも複数のCSRC IDがあるのは？

A. 1つの国に複数の販社・サービス拠点があるため。

## F. ビジネスルール

### Q30. クリックチャージとは？

A. **印刷枚数連動の課金モデル**。販社→コニカミノルタへ「1枚あたりN円」を支払う。
KMの安定収益源。

### Q31. クリックチャージの単価はいくら？

A. **機種・契約により異なる**。研修では仮値 **@3円/枚** で計算しますが、正確な数値は社内確認推奨。

### Q32. 顧客が機械を所有？レンタル？

A. **契約形態により異なる**（買取・リース・レンタル等）。
**現在のデータには契約形態の列はない** ため、必要なら別データソースを取り寄せる必要あり。

### Q33. なぜ売上UPに「印刷枚数」が重要？

A. クリックチャージ収入 = 機体数 × **月平均印刷量** × 単価。
機体数や単価は短期で大きく動かしにくいが、**1台あたりの印刷量を上げる** ことは施策で動かせる。

## G. 困った時のおまじない

### 質問に答えられない時の対応

1. **「データ仕様書を見てみよう」** — 大抵の列定義はここに書いてある
2. **「公式ページで確認してみよう」** — 製品仕様やオプション詳細はweb参照
3. **「社内の○○に確認します」** — 業務ルール・契約・最新機種等
4. **「データから推論してみよう」** — 答えがない時の演習ネタにする

### 講師の心構え

- **答えがわからない＝悪い質問** ではない。むしろデータ理解が深まるきっかけ
- **間違った仮定で進む** より、**「分からない」と認める** ほうが研修品質は上がる
- **公式リンクで補完** することを教えれば、参加者は自走できるようになる
